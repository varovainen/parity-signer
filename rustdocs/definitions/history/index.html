Types used in Signer history log

Signer keeps log of all events in HISTORY tree of the cold database.

Every log Entry consists of timestamp and a set of simultaneously occured events Vec<Event>. Entry is stored SCALE-encoded under key Order. Order is produced from the order the event gets in history log when entered. Order is an addition to the timestamp, and normally arranging entries by the timestamp should coincide with arranging entries by Order.

Log by default starts with database init Entry containing:

  • Event::DatabaseInitiated
  • Event::GeneralVerifierSet(_)

User can clear history log at any time. This indeed will remove all history entries, and the log will then start with Entry containing Event::HistoryCleared.
