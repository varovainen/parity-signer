Definitions and common methods for types used in Signer and Signer-supporting ecosystem.

Features

Feature "signer" corresponds to everything related to Signer air-gapped device.

Feature "active" corresponds to all Signer-related things happening without air-gap.

Feature "test" includes both "signer" and "active" features, along with some testing, and is the default one.
