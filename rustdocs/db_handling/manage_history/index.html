Displaying and updating history log

The Signer keeps a history log of all events that change the database and affect security. It is stored in HISTORY tree of the cold database.

Each history log Entry contains Event set and a timestamp. Database key for Entry value is Order, SCALE-encoded number of the entry.

In addition to keeping the log, Signer also displays HISTORY tree checksum for user to possibly keep the track of.
